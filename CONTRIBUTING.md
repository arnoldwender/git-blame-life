```
git log --oneline CONTRIBUTING.md

```

Welcome. We already know who to blame for this.

## Getting Started

```bash
git clone https://github.com/arnoldwender/git-blame-life.git
cd git-blame-life
npm install
npm run dev
```

The app should be running at `http://localhost:5173`. Start blaming.

## How to Contribute

1. **Fork the repo** and create a feature branch (`feat/new-blame-targets`)
2. **Make your changes** — new life decisions, blame categories, merge conflicts
3. **Test locally** — make sure the blame is properly attributed
4. **Submit a PR** (it will inevitably have merge conflicts with your life plans)

## Code Style

- TypeScript everywhere — we need to know exactly who to blame for each type
- React functional components with hooks
- Tailwind CSS for styling
- Git metaphors should be accurate enough that developers appreciate the double meaning
- Life decisions should be universally relatable — everyone has signed up for a gym in January

## Ideas We Love

- New life decisions to blame people for ("Who approved this haircut?" — `git blame hair.css`)
- More merge conflict scenarios (career vs. hobbies, sleep vs. one more episode)
- Creative revert failures ("Cannot revert: tattoo is already in production")
- New blame targets (past self, future self, that one friend, autocorrect)
- Life PRs that get rejected by reality

## Bug Reports

Open an issue. If the blame wheel somehow lands on the right person, that might actually be a feature.

## Feature Requests

Got a life decision that deserves proper version control? Open an issue with the `enhancement` label. Describe the decision, who's to blame, and why it can't be reverted.

## The Golden Rule

We're all here because we've made questionable life decisions and found it funny to frame them as git history. Keep the humor self-deprecating and universal. The best jokes are the ones where everyone sees themselves in the `git log`.
